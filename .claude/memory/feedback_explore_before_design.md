name: feedback_explore_before_design
description: Always explore the live running system (screenshots, actual UI) before proposing any design mockups - never assume based on code alone
type: feedback
---

When designing new features, ALWAYS explore the current live system first — take screenshots from both mobile and desktop of the running app (e.g., shooter.breezehq.dev) before creating any mockups. Don't assume the current UI state based on code reading alone.

**Why:** The user was frustrated when I presented design mockups without first understanding what the current system actually looks like in production. Code doesn't tell the full story — the live product does.

**How to apply:** Before any UI/UX design work: (1) Visit the live site and take screenshots, (2) Show the user the current state, (3) Design new mockups that build on the actual existing experience rather than assumptions from code.
