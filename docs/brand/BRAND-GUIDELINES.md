# SHOOTER Brand Guidelines

## 🎯 **Brand Identity**

### **Mission Statement**
SHOOTER is an AI-powered code targeting and notification system that bridges the gap between intelligent development monitoring and real-time iOS notifications, empowering developers with precision awareness of their coding environment.

### **Brand Personality**
- **Intelligent**: AI-powered precision and smart automation
- **Professional**: Enterprise-ready reliability and sophistication
- **Futuristic**: Cutting-edge technology with sci-fi aesthetics
- **Precise**: Accuracy in targeting and notification delivery
- **Trustworthy**: Dependable monitoring and consistent performance

## 🎨 **Visual Identity**

### **Primary Logo**
**Master File**: `assets/icons/shooter-icon-master.svg`

The SHOOTER logo features a holographic radar interface representing AI-powered code scanning and targeting capabilities.

### **Logo Elements**
1. **Central Radar Display**: Core scanning system (320px radius)
2. **Rotating Sweep Beam**: Continuous monitoring indicator
3. **Holographic Data Points**: Code file representations
4. **Status Panels**: Real-time system indicators
5. **Grid System**: Technical precision overlay

### **Logo Variations**
- **Full Color**: Primary version with complete holographic effects
- **Simplified**: Reduced detail for smaller applications
- **Monochrome**: Single color versions for special contexts
- **Circular**: Cropped version for profile pictures and avatars

## 🌈 **Color Palette**

### **Primary Colors**
- **Holographic Blue**: `#00aaff` - Primary brand color
- **Cyan Highlight**: `#00ffff` - Accent and emphasis
- **Deep Blue**: `#0088cc` - Secondary brand color

### **Background Colors**
- **Space Black**: `#000a1a` - Primary dark background
- **Deep Space**: `#0a1a2a` - Secondary dark background
- **Panel Dark**: `#001122` - UI panel backgrounds

### **Functional Colors**
- **Success**: `#00ff80` - Positive states, confirmations
- **Warning**: `#ffaa00` - Alerts, pending states
- **Error**: `#ff4444` - Errors, critical states
- **Neutral**: `#888888` - Secondary text, inactive states

### **Color Usage Guidelines**
- **Primary Blue** for main brand elements, CTAs, and highlights
- **Cyan** for accents, data visualization, and interactive elements
- **Dark backgrounds** to maintain sci-fi aesthetic and ensure contrast
- **Functional colors** sparingly for status indication only

## 📝 **Typography**

### **Primary Typeface**
**Monospace Fonts** (in order of preference):
1. `SF Mono` (Apple platforms)
2. `Monaco` (macOS)
3. `Cascadia Code` (Windows)
4. `Roboto Mono` (Android/Web)
5. `monospace` (fallback)

### **Typography Usage**
- **Headers**: Monospace, bold weight, holographic blue color
- **Body Text**: Monospace, regular weight, white/light gray on dark
- **Code**: Monospace, cyan highlights for syntax
- **UI Labels**: Monospace, small size, consistent with technical aesthetic

### **Text Guidelines**
- Always use monospace fonts to maintain technical/coding aesthetic
- Ensure high contrast ratios (minimum 4.5:1) on dark backgrounds
- Use cyan highlights sparingly for emphasis
- Maintain consistent spacing and alignment

## 📱 **Logo Usage Guidelines**

### **Minimum Sizes**
- **Digital**: 16x16 pixels minimum
- **Print**: 0.5 inches (12.7mm) minimum
- **Business Cards**: 0.25 inches (6.35mm) minimum

### **Clear Space**
Maintain clear space around the logo equal to 1/4 of the logo's width on all sides.

### **Acceptable Uses**
✅ Use on dark backgrounds for optimal contrast  
✅ Scale proportionally (maintain aspect ratio)  
✅ Use official color versions when possible  
✅ Apply to merchandise with brand approval  
✅ Use in presentations and documentation  

### **Prohibited Uses**
❌ Don't change colors or gradients  
❌ Don't add effects, shadows, or outlines  
❌ Don't stretch or distort proportions  
❌ Don't use on light backgrounds without adaptation  
❌ Don't combine with competing visual elements  
❌ Don't use low-resolution versions at large sizes  

## 🖥️ **Digital Applications**

### **Website & Web Apps**
- Use animated SVG version for hero sections and primary brand placement
- Static PNG versions for navigation and secondary placements
- Ensure responsive scaling across all device sizes
- Maintain minimum size requirements for mobile devices

### **iOS Applications**
- Use platform-specific icon set from `assets/icons/ios/`
- Follow Apple Human Interface Guidelines
- Ensure clarity at all required sizes (20pt to 1024pt)
- Use appropriate versions for different contexts (app icon, notifications, settings)

### **Social Media**
- Use dedicated social media assets from `assets/icons/social/`
- Platform-specific optimizations for profile pictures and covers
- Maintain brand consistency across all social platforms
- Use circular versions where platforms auto-crop to circles

## 🎬 **Animation Guidelines**

### **Logo Animation**
- **Radar Sweep**: 4-second rotation cycle, smooth and continuous
- **Data Materialization**: 3-second fade sequence for data points
- **Status Pulse**: 2-second breathing animation for indicators
- **Particle Float**: 4-second vertical movement for ambient elements

### **Animation Usage**
- Use animations for web headers, loading states, and brand showcases
- Static versions for contexts where animation is inappropriate
- Ensure animations enhance rather than distract from content
- Consider accessibility and motion sensitivity preferences

## 📄 **Print Applications**

### **Business Materials**
- Use high-resolution PNG or vector formats
- Ensure minimum size requirements are met
- Maintain color accuracy with Pantone equivalents when possible
- Consider monochrome versions for cost-effective printing

### **Merchandise**
- Approve all merchandise applications through brand review
- Ensure logo quality and placement maintains brand integrity
- Use appropriate file formats for different printing methods
- Test visibility and impact at actual product sizes

## 🔧 **Technical Specifications**

### **File Formats**
- **SVG**: Master file and web applications
- **PNG**: Digital applications with transparency
- **ICO**: Windows applications and favicons
- **ICNS**: macOS application bundles

### **Color Profiles**
- **RGB**: Digital applications (sRGB color space)
- **CMYK**: Print applications (convert from RGB)
- **Pantone**: High-end print and merchandise (closest matches)

### **Resolution Standards**
- **Web**: 72 DPI minimum, optimized file sizes
- **Mobile**: 2x and 3x densities for Retina displays
- **Print**: 300 DPI minimum for professional quality

## 📁 **Asset Organization**

### **File Structure**
```
assets/icons/
├── shooter-icon-master.svg (source file)
├── ios/ (complete iOS icon set)
├── web/ (favicons and web assets)
```

### **Naming Conventions**
- Use descriptive, lowercase names with hyphens
- Include size specifications where relevant
- Specify platform or usage context
- Version control with clear file organization
